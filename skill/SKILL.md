# Daily FlipBook Generator

Скилл, который читает журнал сессий Claude Code за сегодня и генерирует дневной обзор в виде «книги» с перелистыванием страниц.

## Триггер

- Запуск по `/daily-flipbook`
- «Создай книгу-обзор за сегодня», «Сгенерируй книгу-дневник» и т.п.

## Порядок обработки

### Step 1: Сбор журнала сессий

Собрать журнал сессий Claude Code за сегодня из обоих путей:

```bash
# Путь 1: по умолчанию
~/.claude/projects/

# Путь 2: указанный через CLAUDE_CONFIG_DIR
~/claude-data/projects/
```

**Обязательно проверять оба пути. Нельзя говорить «не найдено», проверив только один.**

Файлы логов в формате JSONL. Используйте следующую команду для поиска сегодняшних сессий:

```bash
# Поиск JSONL-файлов, обновлённых сегодня
find ~/.claude/projects/ -name "*.jsonl" -newer /tmp/today_marker -not -path "*/subagents/*" 2>/dev/null
find ~/claude-data/projects/ -name "*.jsonl" -newer /tmp/today_marker -not -path "*/subagents/*" 2>/dev/null
```

Создание маркера даты:
```bash
touch -t $(date +%Y%m%d)0000 /tmp/today_marker
```

### Step 2: Анализ логов

Из каждого JSONL-файла извлечь следующее:

1. **Сообщения пользователя** (`type: "user"`) → что было запрошено
2. **Ответы ассистента** (`type: "assistant"`) → что было выполнено
3. **Использование инструментов** (`tool_use`) → содержимое выполнения Write, Edit, Bash и т.д.

Упорядочить по хронологии и структурировать следующим образом:

```
- Сессия 1: [Название проекта]
  - Что сделано: [Краткое описание]
  - Результаты: [Файлы, коммиты и т.д.]
  - Выводы: [Наблюдения, открытия]

- Сессия 2: ...
```

### Step 3: Определение структуры книги

Автоматически сгенерировать структуру книги из собранной информации:

| Страница | Содержание |
|----------|------------|
| Обложка | Дата + Заголовок («Записи за ДД.ММ.ГГГГ») |
| Введение | Сводка за день (количество сессий, основные результаты) |
| Главы | Подробности по каждой сессии (что сделано, код, выводы) |
| Последняя страница | Обзор дня + Планы на завтра |

**Правила структуры:**
- Одна сессия — один разворот (2 страницы)
- Максимум 6 глав (12 страниц). При большем количестве отбирать по важности
- Использовать блоки кода, цитаты и блоки советов по необходимости

### Step 4: Генерация FlipBook HTML

Сгенерировать HTML на основе шаблона (`index.html`), заменив массив `pages`.

Путь вывода: `/tmp/claude/daily-flipbook/YYYY-MM-DD.html`

Доступные CSS-классы:
- `.page-title` — основной заголовок
- `.chapter-label` — номер главы
- `.page-body` — основной текст
- `.code-block` — код (подсветка через `.comment`, `.keyword`, `.string`, `.property`)
- `.quote` — блок цитаты
- `.tip-box` — блок советов (`.tip-title` + `<ul>`)
- `.comparison` — сравнение в 2 колонки (`.col.good` / `.col.bad`)
- `.divider` — декоративная разделительная линия
- `.dropcap` — буквица (`<p class="dropcap">`)

### Step 5: Открыть в браузере

```bash
open /tmp/claude/daily-flipbook/YYYY-MM-DD.html
```

## Пример вывода

Обложка:
```
Записи за 5 апреля 2026 года
── Today I Learned ──
```

Пример главы:
```
Глава первая: Создал UI FlipBook

Началось с вопроса «Можешь сделать UI с перелистыванием страниц?»,
и в итоге с помощью CSS 3D transforms и Web Audio API
был создан интерфейс книги с перетаскиванием страниц.

По ходу дела добавилось требование
«чтобы можно было по-настоящему перелистывать мышью»,
и пришлось реализовать обработку перетаскивания.
mousedown → mousemove → mouseup для
вычисления угла: если больше 50 градусов — завершить,
если меньше — вернуть пружиной назад.
```

## Примечания

- Если журнал сессий не найден, сообщить «Сегодня сессий ещё не было»
- Если в логах содержится персональная информация (API-ключи, пароли и т.д.), исключить её
- Сгенерированный HTML также не должен содержать персональную информацию
- Для анализа логов использовать Bash-инструмент (jq)
